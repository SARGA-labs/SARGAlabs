# SARGA SYSTEM (LLM.txt)

> SARGA 2026. A system for making.

## WHAT IS SAR.GA

SARGA is art, code, sound—one system, infinite outputs.

We build like engineers. We ship like a label. Same engine, different releases.

**NOT:** An agency. A collective. A magazine.
**YES:** A modular studio. A publishing system. A frequency.

**Aesthetic DNA:** Chrome Hearts brutalism + Swiss grid discipline
**Tone:** Direct. No filler. Declarative statements. Specs over stories.

## VISUAL SYSTEM

### COLOR SYSTEM

**Base (90-95% of all surfaces):**
```css
--ink: #000000
--paper: #ffffff
```

**Signal (identity indicators, rare usage):**
```css
--signal-sarga: #0000ff      /* SARGA system */
--signal-diyuksh: #ff0000    /* DIYUK.SH studio */
--signal-notebook: #00ff00   /* DIYUKSH BY DIYUKSH */
```

**Signal usage rules (CRITICAL):**
- Only ONE signal element per layout/screen/post
- Never as background fills (only as strokes, underlines, dots, badges)
- Think LED status indicator, not "brand color"
- 2px maximum width for rules/underlines
- Common uses:
  - Active state underline in capsule
  - 2px corner rule (top-left or bottom-right)
  - Single dot bullet (4px × 4px)
  - Thin border on hover/active state
  - Icon fill (rare)

**What NOT to do:**
```css
/* WRONG - too much color */
background: #0000ff;
border: 4px solid #0000ff;
box-shadow: 0 0 20px #0000ff;

/* RIGHT - signal as accent */
border-bottom: 2px solid #0000ff;
/* or */
background: #fff;
border: 1px solid #000;
border-left: 2px solid #0000ff; /* tiny signal */
```

**Weight system:**
```css

--weight-bold: 600;     /* rare, only for emphasis */
```

**Letter spacing (tracking):**
```css
--tracking-tight: -0.02em;    /* large display type only */
--tracking-normal: 0;         /* default, most text */
--tracking-wide: 0.05em;      /* ALL CAPS labels */
--tracking-wider: 0.1em;      /* SPEC LINES */
```

**Line height:**
```css
--leading-tight: 1.2;    /* headlines */
--leading-normal: 1.5;   /* body */
--leading-loose: 1.75;   /* long-form */
```

**Typographic rules:**
1. Tight tracking ONLY on display sizes (48px+)
2. ALL CAPS always gets wider tracking (0.05em minimum)
3. Body text always `--tracking-normal` (0)
4. Use weight contrast (400 vs 500) not size variation
5. Never use more than 3 sizes per screen/page
6. Spec lines always in mono at `--text-xs`

## COPY SYSTEM

### VOICE + TONE

**Core principles:**
1. **Direct.** No preamble. Start with the point.
2. **Declarative.** Statements, not questions. No "maybe" or "might."
3. **Minimal.** If you can cut a word, cut it.
4. **Specific.** Numbers, specs, materials. No vague adjectives.
5. **Confident.** We know what we're doing. No apologies.

**Chrome Hearts / Off-White influence:**
- Diagonal slashes in copy: SARGA / LABS / BUILDS
- Specs as poetry: model numbers, versions, timestamps
- Material callouts: 100% COTTON / 280GSM / REINFORCED STITCHING
- Instructions as design: "FOR BEST RESULTS" "USE AS DIRECTED"

**What we sound like:**
- "Built like software. Felt like music."
- "Signal over noise."
- "Structure over trend."
- "High fidelity, low light."
- "Performance measured."
- "Everything tuned."

**What we DON'T sound like:**
- "We're passionate about creating experiences..."
- "Join us on a journey..."
- "Crafted with love and attention to detail..."
- "Your one-stop shop for..."
- "Elevating the art of..."
